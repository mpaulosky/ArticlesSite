## Technical Notes

XML parsing in centralized package management files is all-or-nothing. When a syntax error occurs, the parser stops processing at that point. This creates a cascading failure where one typo can prevent package version resolution for dozens of packages downstream in the same file.

**Always verify:**
- XML syntax validity before committing centralized config files
- Dependency version alignment (MongoDB.Driver ↔ MongoDB.Driver.Core)
- Build cache files (.lscache, bin/, obj/) are in .gitignore
